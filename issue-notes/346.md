# issue rキーのランダム音色は、UX検証用の仮仕様として、現在のpatchカテゴリ例えば pad 等のなかからランダムで選ぶようにする #346
[issues #346](https://github.com/cat2151/clap-mml-render-tui/issues/346)

- これまでの課題： Notepadモードで、rキーを押すたびに、すべての音色からランダムで選ばれていた。これを、basses カテゴリの音色に対してrキーを押したら、basses カテゴリだけを対象にランダムで選ぶ（いろいろなbasses音色をランダムで選べる）という体験がほしい、と感じた。
- 対策：rキーのランダム音色は、「jsonでfilter指定がなく、かつjson patch nameがある場合」、現在のpatchカテゴリ例えば pad 等のなかからランダムで選ぶようにする
    - notepadモードで、一度この機能を発動させたら、jsonのpatch name filterにも（dawモード同様）patchカテゴリ名をフィルタとして登録する
    - そもそも現在行にpatch nameがなくpatchカテゴリが得られない場合は、従来通りの挙動とする（全ての音色からランダムで選ぶ）
    - dawモードで、patch name filterがinit measによって指定済みの場合は、従来通りとする（現在のpatchカテゴリからでなく、filterで絞ったlistからランダムで選ぶ）
    - notepadモードでも、dawモード同様、patch name filterがinit measによって指定済みの場合は、それを優先する（現在のpatchカテゴリからでなく、filterで絞ったlistからランダムで選ぶ）
