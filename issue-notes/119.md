# issue patch nameごとのhistoryとお気に入りを実装する #119
[issues #119](https://github.com/cat2151/clap-mml-render-tui/issues/119)

- userがほしいもの
    - あるpatchを選んだとき、演奏した過去の短いフレーズで、そのpatchのスイートスポットをすぐ把握したい
    - 対策
        - そのための画面と仕組みを検証しながら構築していく
            - 小さく始める
- 前提
    - 先頭にpatch name jsonがあるときにそれをpatchとして鳴らす、という仕組みになっている
        - 確認済み
- 備忘
    - おそらくこのissueによるPRは、一部を実装漏れする
        - なぜなら、
            - ここまで大きな実装を一度のPRでcoding agentはできない可能性が高い、
                - 現時点ではagentの力不足
        - 対策
            - 未実装があれば、
            - このドキュメントを軸に、
            - 切り分けて、
            - 未実装のものを
            - 1つずつ別issueで実装していく
- 方法
    - 新規json
        - patch history json
            - patch nameがkey、
            - valueに、historyとお気に入りを持つjson
            - enterやescで確定するごとに保存、起動時に復帰
    - 新画面
        - patch phrase 画面
            - patchごとのhistoryとお気に入りを管理する
    - 記録機能
        - patch ごとのhistory記録機能
            - 通常の非DAW画面において、行頭にpatch name jsonがある場合は、
                - 演奏時、その行をpatch historyにも記録する
                    - patch historyのデータ構造は別項を参照
    - UI
        - pキーで、patch phrase 画面にいくかの判定
            - 今の行頭にjsonがあり、そこにpatch nameがあるかどうかで分岐
                - ない場合は、エラー表示
                - ある場合は、patch phrase画面に遷移する
        - patch phrase画面
            - 今のMMLのpatchに対する、
                - history list を表示
                    - 左ペイン。
                - お気に入り list を表示
                    - 右ペイン。
                - どちらも、listが0件の場合は、`c` だけをlistに1件挿入しておく
            - jkキーで上下移動しつつ演奏
            - hlキーでペイン移動しつつ演奏
            - fキーで、その行をお気に入り先頭に追加
            - ESCキーで非daw画面に戻る
            - ※小さく始めるため、iキーなし。別issueで追加とする
- 参考、実績、easymmlabcリポジトリ
    - demoで、GM音色0～127とdrumのphrase examplesを
        - 軽量MMLで作成した実績がある
