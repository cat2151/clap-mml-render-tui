# issue notepadとhistory系は、予測cacheを実装する #323
[issues #323](https://github.com/cat2151/clap-mml-render-tui/issues/323)

- これまでの課題：
  - jkキーで音を鳴らしながら移動しているとき、jやkを押してからオフラインレンダリングが走るため、押してから鳴るまでに時間がかかることが多く、UXが悪い
  - pageUp, pageDownも同様
- 対策：
  - 現在行で音を鳴らしたなら、
  - 次に、j,k,pageUp,pageDownキーが押されると予測し、次にそれらのキーが押されたときの移動先の行をオフラインレンダリングしてcacheに入れておく
