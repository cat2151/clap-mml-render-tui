# issue DAWモード、meas2～8が空欄にも関わらず、meas1～8をloop演奏してしまっている #117
[issues #117](https://github.com/cat2151/clap-mml-render-tui/issues/117)

- userがほしいもの
    - meas2～8が空欄なら、meas1のみをloop演奏
- これまでの流れ
    - 実装させたときのsessionをみた
        - coding agentは、「実装済み」とソースをみて判断している
    - その判断は、誤りである
- 分析
    - agentが現実を見ることができていない
        - 検証せず、机上ソースチェックだけで判断している
            - 典型的なagentのアンチパターンの挙動である
- 対策
    - agentに現実を見せる
    - つまり、問題をagentにもわかるよう可視化させる
        - 表示
            - 追加で1行、表示欄をつける。表示内容は：
                - loop meas : meas 1, empty meas : meas 2～8
        - log
            - 以下を、play開始時にlogに出力させる：
                - meas1 : 内容があります
                - meas2 : empty
                - :
                - meas8 : empty
                - 有効meas : meas1
                - empty meas : meas 2～8
                - loop start meas : meas1
                - loop end meas : meas1
    - agentにtestを書かせる
        - この機能自体はtestにできるはず
        - testが既にあって誤っている可能性も考慮しておく
